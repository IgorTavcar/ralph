# Example: Audio Chaptering Tool

Full end-to-end example: interactive planning via `ask_user_questions` MCP,
then autonomous implementation via Ralph.

**Topic:** A CLI tool that generates chapter markers for audio files
(podcasts, audiobooks, lectures) using transcription + AI summarization.

## Prerequisites

- [bun](https://bun.sh) installed
- [Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code) installed
- `ANTHROPIC_API_KEY` set

## Two-Phase Workflow

### Phase 1: Plan (interactive)

Claude interviews you in depth via the AUQ terminal UI, then writes the PRD.

```bash
# Terminal 1 — start planning
export ANTHROPIC_API_KEY=sk-ant-...
./plan.sh

# Terminal 2 — answer questions here
bunx -y auq-mcp-server
```

Claude asks 3-5 rounds of deep questions about architecture, UX, tradeoffs,
edge cases. After the interview it writes:
- `tasks/prd-audio-chaptering.md` — human-readable PRD
- `prd.json` — Ralph-format task list

### Phase 2: Build (autonomous)

Ralph implements the stories from `prd.json` one by one.

```bash
# Same terminal, after plan.sh finishes
./build.sh
```

This copies `ralph.sh` locally and runs the loop. Each session:
1. Picks the next incomplete story
2. Implements it
3. Runs quality checks
4. Commits if passing
5. Marks the story done in `prd.json`
6. Repeats until all stories pass

## Files

| File | Phase | Purpose |
|------|-------|---------|
| `plan.sh` | Plan | Starts the interview session |
| `PLAN.md` | Plan | Interview instructions + PRD generation guide |
| `.mcp.json` | Plan | Wires up `ask_user_questions` MCP server |
| `.auqrc.json` | Plan | AUQ display/behavior config |
| `build.sh` | Build | Runs Ralph to implement stories |
| `CLAUDE.md` | Build | Standard Ralph agent instructions |
| `prd.json` | Both | Generated by planning, consumed by Ralph |
| `tasks/` | Plan | Output directory for the markdown PRD |
